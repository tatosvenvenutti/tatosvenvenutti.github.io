Gameplay Logic

As gameplay programmer for this project my goal was enemy logic such as movement, player controls such as movement, firing, and inputs. As well as gameplay loop, and prefabs spawning and collision logic.

  • Implemented enemy movement and spawning
  • Programmed character controls and projectiles
  • Generated premade assets as prefabs

Creating a custom engine cause a lot of issues for our team, so we decided to take our insperation from Space Invaders.

Enemy Logic

The EnemyLogic function handles enemy behavior in the game by processing each enemy entity for collision and movement. If an enemy collides with something, it is removed from the game. Otherwise, the function updates its position based on defined movement logic. This ensures dynamic and responsive enemy actions during gameplay.

  • Collision Detection to removes enemies that collide with other objects
  • Updatingp enemy positions for both horizontal and vertical movements
  • Processed each enemy entity using ECS (Entity Component System) architecture.

Player Controls

The PlayerLogic function is a player responsive input handling for gameplay. This function also governs shooting mechanics, regulating the firing rate and coordinating shooting actions based on player input.

  • Playerinteractions, including movement and shooting mechanics
  • Utilized timing calculations and responsive input handling for smooth player control
  • modelling transformations and collision detection

Entity Component System

The Entity-Component-System (ECS) architecture modular and efficient in game development. I created component functions for enemies and projectiles.

  • Utilizes ECS for prefab management
  • Employs flexible entity creation and behavior customization components
  • Generated standardized entity templates (prefabs)
