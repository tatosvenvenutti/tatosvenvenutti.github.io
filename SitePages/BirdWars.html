Gameplay Logic

As a gameplay programmer for this project my focus was implementing core gameplay systems, including enemy behavior, player movement, firing mechanics, input handling, and collision systems.

  • Implemented enemy movement and spawning
  • Programmed character controls and projectiles
  • Generated premade assets as prefabs

Reference and Inspiration

Developing a custom engine introduced several technical challenges, so we drew inspiration from Space Invaders..

Enemy Logic

The EnemyLogic function processes enemy entities, handling both movement and collision. Enemies are removed upon collision, otherwise their positions are updated based on movement behavior, ensuring responsive and dynamic gameplay.

  • Collision detection that removes enemies that collide with other objects
  • Updating enemy positions for both horizontal and vertical movements
  • Processed each enemy entity using ECS (Entity Component System) architecture.

Player Controls

The PlayerLogic function handles responsive player input and core gameplay interactions. This function also governs shooting mechanics, regulating the firing rate and coordinating shooting actions based on player input.

  • Player interactions, including movement and shooting mechanics
  • Utilized timing calculations and responsive input handling for smooth player control
  • Handling transformations and collision detection

Entity Component System

The Entity-Component-System (ECS) architecture is modular and efficient for game development. I implemented component-based systems for enemies and projectiles using ECS architecture.

  • Utilizes ECS for prefab management
  • Employs flexible entity creation and behavior customization components
  • Generated standardized entity templates (prefabs)
